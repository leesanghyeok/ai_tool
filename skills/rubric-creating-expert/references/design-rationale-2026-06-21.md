# rubric-creating-expert 설계 근거 — 2026-06-21

## 사용자 교정에서 나온 durable lesson

- 이 스킬은 루브릭 작성 원론을 반복하지 않는다. 기본 루브릭 설계는 `rubric-design`에 위임한다.
- 스킬의 책임은 “세계 최고 수준 전문가 루브릭을 실제 신규 답변으로 캘리브레이션하는 파이프라인”이다.
- 실행 언어는 항상 한국어로 고정한다. 별도 언어 변수는 두지 않는다.
- 도메인이 명확하면 그대로 진행하되, 애매하면 추측하지 말고 사용자에게 확인한다.
- 목표 평균점수, 질문 수, 최대 버전, artifact 경로는 루브릭 채점표의 관심사가 아니다. 반드시 `run-config.md` 같은 워크플로우 설정 파일로 분리한다.
- 루브릭 파일에는 평가 관심사만 둔다: 평가 목적, 대상, 전문가 기준, 배점, cap/gate, judge 지침, 점수 해석 등.

## 개선된 병렬 검증 파이프라인

기본 루프:

1. 질문 생성 에이전트가 N개의 새 질문을 생성한다.
2. 질문별 신규 서브에이전트 N개가 병렬로 다음을 수행한다.
   - 루브릭을 보지 않고 질문에 대한 일반 답변 작성
   - 답변 원문 파일 저장
   - 그 다음 현재 루브릭으로 자기 답변 채점
   - 채점 결과 파일 저장
3. 부모 에이전트가 파일 존재, `final_score`, 평균 계산을 직접 검증한다.
4. 평균이 `target_average` 미만이면 종료한다.
5. 미달성 시 고득점 답변의 과대평가 원인을 분석해 루브릭을 개선한다.
6. 다음 루프에서는 질문을 재사용하지 않고 새 질문 세트를 생성한다.

## holdout validation 처리

이 워크플로우에서는 매 루프마다 질문이 새로 생성되므로 기존 답변에 대한 과적합이 반복되기 어렵다. 따라서 별도 holdout validation은 기본 단계에서 제외한다. 디버깅이나 비교 목적일 때만 예외적으로 기존 질문 재채점을 허용한다.

## 부모 에이전트 책임

서브에이전트의 “완료” 보고는 검증된 사실이 아니다. 부모 에이전트가 반드시 다음을 직접 확인한다.

- 응답 파일 N개 존재
- 점수 파일 N개 존재
- 각 점수 파일에 `final_score` 존재
- 평균 계산은 도구로 수행
- 목표 평균 달성 여부 판단
